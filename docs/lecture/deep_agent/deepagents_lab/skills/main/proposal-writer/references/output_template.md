# Preferred proposal structure

Use this structure unless the user requests another format:

## Proposed title

## Executive summary and context

## Problem and motivation

## What the team wants to build

## Data, resources, and recent related work

## Planned methods and multi-agent workflow

## Evaluation plan

## Risks and open questions

## References and source information

Notes:

- Write the main body as a compact executive summary of about three pages.
- Use prose paragraphs under each heading instead of bullet dumps.
- The proposal should sound like something a student team could realistically execute.
- Mention only current, evidence-backed benchmarks and systems.
- Use recent scientific writing to make the proposal concrete and current.
- The evaluation plan should include at least one baseline or comparison point.
- End with a short references section that lists only the sources actually used in the writing.
- Include enough source information to identify the item again later: title, source or organization, year if available, and URL.
