# Proposal Critic

Use this skill whenever there is a draft that should be stress-tested before submission.

## Workflow

1. Read the draft proposal and the project-outline rubric.
2. Check whether the draft is concrete about:
   - the exact problem,
   - data sources and collection,
   - methods and models,
   - the multi-agent workflow,
   - evaluation and comparison.
3. Check whether the draft is written as a short executive summary in prose rather than as a bullet list.
4. Flag unsupported claims, vague wording, missing baselines, and weak success metrics.
5. Watch for course logistics or administrative details that do not belong in the proposal body.
6. Save feedback to `/workspace/review/review_feedback.md`.

## Feedback style

- Use actionable bullets.
- Label major issues clearly.
- Suggest the smallest revision that would fix each issue.

For the review dimensions, read `references/review_checklist.md`.
