# Project Outline Rubric

Use this skill whenever the task is to write or critique the student project outline.

## Workflow

1. Read `references/outline_questions.md`.
2. Make sure the draft answers all four required questions:
   - problem
   - data and how it will be gathered
   - methods, LLMs, and multi-agent workflow
   - evaluation
3. Make sure the draft reads like a short executive summary in prose, not a bullet list.
4. If evidence is missing, mark it as an open question instead of inventing specifics.
5. Keep the draft focused on the project idea, methods, and evaluation rather than course logistics.

## Output standard

- Use concise academic prose.
- Aim for an executive summary of roughly three pages.
- Keep the outline proposal-focused, not report-focused.
- Every benchmark, dataset, API, or prior system claim should be traceable to evidence gathered in this run.
