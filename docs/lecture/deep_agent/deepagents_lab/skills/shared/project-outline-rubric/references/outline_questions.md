# IE686 project outline questions

Use these questions as the required structure for the proposal:

1. What is the problem you are solving?
2. What data will you use?
   - Where will you get it?
   - How will you gather it?
3. How will you solve the problem?
   - What LLMs do you plan to use?
   - Which methods do you plan to apply?
   - What is your idea for a multi-agent workflow for your task?
4. How will you measure success?

Additional guidance:

- Treat this as a project outline or proposal draft, not as the final report.
- Write it as a compact executive summary in prose rather than as a bullet list.
- A strong draft usually reads like roughly three pages of content.
- Strong proposals include concrete datasets, benchmarks, or APIs, at least one comparison point, and a measurable evaluation plan.
